SDErasure (ICLR 2026)

Concept erasure via adaptive denoising trajectory shifting

SDErasure Overview

SDErasure is a fine-tuning method. We borrow its SSScore idea for inference-time use.

SSScore — Step Separability Score

Quantifies per-timestep concept discriminability

St = 1 / (1 + exp(Lc − La))
Lc = ||εθ(xt, ctarget, t) − ε||²
La = ||εθ(xt, canchor, t) − ε||²
Interpretation:
  • St ≈ 1.0 → Model recognizes target at step t
  • St ≈ 0.5 → No discrimination → skip
  • St ≈ 0.0 → Anchor dominates
Key property: SSScore tells us exactly which timesteps the model "decides" to generate the target concept. SDErasure applies fine-tuning loss only at high-SSScore timesteps.
Requirement: Semantic anchor is essential. Empty anchor → St ≈ 0.5 everywhere (no signal).

Part 2

SGF — Safety-Guided Flow

ICLR 2026 Oral · A unified framework for negative guidance

SGF Overview

Kim et al. — Unified framework showing SPELL and Safe Denoiser are special cases

Core Mechanism

x̃t = fθ(xt, t) + λ(t) · ∇x E(xt)
E(xt) = MMD²(kσ)({xt}, D−)
MMD gradient as repulsive force: Push generated samples away from an unsafe reference dataset D− using Maximum Mean Discrepancy.

Key Contributions

  • Unification: SPELL, Safe Denoiser are special cases of MMD guidance
  • CBF Theory: Control Barrier Function analysis proves earlier timestep guidance is strictly better
  • Plug-and-play: Compatible with SLD, SAFREE, etc.
  • Lightweight: +0.10s/image overhead
Limitation: SGF applies guidance globally — no spatial control. Safe regions are also modified.

How SGF Relates to Our Approach

SGF provides:
CBF-proven time window + MMD repulsion (undirected, global)
Our approach adds:
CAS-based selective trigger (WHEN) + GradCAM spatial mask (WHERE) + directed classifier gradient (HOW)
Key difference: SGF modifies all images globally. We only intervene when CAS detects harmful content, and only in harmful spatial regions.

Part 3

Concept Alignment Score (CAS)

Online adaptation of SDErasure's SSScore idea

From SSScore to Concept Alignment Score

Key insight: replace offline precomputation with online cosine similarity

SDErasure SSScore (Offline)

St = 1 / (1 + exp(Lc − La))
  • Precomputed on random latents
  • Not prompt-specific
  • On vanilla SD: St ≈ 0.5 ∀ t
  • Designed for fine-tuning loss weighting
→

Concept Alignment Score (Online)

CASt = cos(dprompt, dtarget)
  • Computed on actual generation latents
  • Prompt-specific: high for harmful, low for safe
  • Works on vanilla SD out of the box
  • Designed for inference-time "when" detection
Shared insight: Both measure how much the model's behavior at timestep t relates to the target concept. CAS makes this prompt-aware by using the actual generation's noise prediction direction.

How CAS Works

zt

UNet(zt, "nudity")

dprompt = εp − ε∅

CAS = cos(dp, dt)
> τ → trigger
dprompt = ε(prompt) − ε(∅)
The direction the current prompt pushes generation. Already computed in CFG — free.
dtarget = ε("nudity") − ε(∅)
The direction the target concept would push. Costs 1 extra UNet forward per step.
Harmful prompt (Ring-A-Bell):
dprompt aligns with d"nudity"
→ CAS ≈ 0.6–0.9 → trigger guidance
Safe prompt (COCO):
dprompt orthogonal to d"nudity"
→ CAS ≈ 0.1–0.3 → skip guidance
Sticky mode: Once CAS exceeds τ at any timestep, guidance remains active for all subsequent steps.

Why CAS Achieves Selectivity

The key difference from SSScore and other monitors

1
Prompt-specific signal
Unlike SSScore (same schedule for all prompts), CAS measures the actual generation direction at each step against the target concept.
2
Directional, not magnitude-based
Grad Norm and Noise Div measure magnitude — high for any prompt with strong signal. CAS measures direction — high only when the prompt's noise direction aligns with the target concept.
3
No fine-tuning required
Works on vanilla SD v1.4. The cosine similarity between noise prediction directions is inherently informative about concept alignment, even without model modification.

Experimental Results

TP/FP selectivity, COCO quality impact, and grid search

Spatial Classifier Guidance (WHERE + HOW)

CAS answers "when" — spatial CG handles "where" and "how"

WHERE: GradCAM Spatial Mask

M(p) = 1[ Φ((Hp − μ) / σ) ≥ τ(t) ]
GradCAM heatmap from a latent-space 4-class classifier (benign / safe_clothed / harm_nude / harm_color) → z-score → CDF → binary mask.
Only harmful spatial regions receive guidance.

HOW: Bidirectional Gradient

gt = ∇z log p(safe | zt) − ∇z log p(harm | zt)
Simultaneously attract toward safe_clothed (class 1) and repel from harmful classes (2, 3).
Injected into noise estimate, weighted by spatial mask.
ε̂t = εθ(zt, t, p) − 1[CASt > τ] · √(1 − αt) · [ Mt · sg + (1−Mt) · sb ] · gt
