Part 2

v4–v5: Foundation

Online CAS + SLD/Anchor Guidance

v4 Anchor Inpainting

Baseline method: all 4 UNet passes online

Architecture

WHEN: Online CAS (cos(dp, dt) > τ)
WHERE: Noise-based spatial CAS (3×3 neighborhood)
HOW: Anchor inpaint / SLD / Hybrid modes
Anchor Inpaint:
εfinal = εcfg(1 − s·M) + εanchor_cfg(s·M)
Hybrid:
εfinal = εcfg − ts·M·dtarget + as·M·danchor

Key Properties

  • 4 UNet calls/step: ε∅, εprompt, εtarget, εanchor
  • All directions computed online (image-specific)
  • Text-based anchor: "clothed person"
  • CAS threshold τ = 0.6
Limitation: 4 UNet passes = expensive. Text anchor is generic, not grounded in real images.
ConfigNudeNet↓Note
v4 hybrid ts10_as15~27.5%Baseline
v4 baseline (no guide)75.6%No safety

v5 Hybrid Projection (hybrid_proj)

Project out nudity component + anchor blend

Innovation

Step 1: dsafe = dprompt − proj(dprompt, dtarget)
Step 2: εsafe = ε∅ + cfg · dsafe
Step 3: εfinal = εsafe(1−a·M) + εanchor(a·M)
Key idea: Instead of subtracting the full target direction, project it out from the prompt direction. This preserves non-nudity content.

Results

Moderate improvement over v4 anchor inpaint, but global projection (not per-pixel) limits precision.
Insight: hybrid mode (subtract + anchor) outperforms hybrid_proj. Direct guidance is stronger than projection at this stage.

v6–v7: Exemplar-Based

Cross-Attention WHERE + GLASS Exemplar Anchor

v6 Cross-Attention WHERE

Semantic spatial masking via attention probing

Innovation: Attention-based Spatial Mask

Probe Mechanism:
  • Inject probe into cross-attention layers (res 16, 32)
  • Q = image features, K = pre-computed target keys
  • Probe attention = where "nudity" tokens attend
  • Zero extra UNet calls (just one matmul/layer)
Token matching: Find nudity-related tokens in prompt ("nude", "naked", "bare", ...) → aggregate attention for those positions

v7 GLASS Exemplar Anchor

Pre-computed anchor from diverse clothed exemplars — best performer

Key Innovation

Offline preparation (one-time, ~10 min):
  • Generate 16 nude + 16 clothed exemplar images
  • Forward-diffuse each to zt
  • Compute per-timestep concept directions
  • Average across exemplars → robust anchor direction
Generation (3 UNet calls/step):
  • Pass 1&2: ε∅, εprompt (batched) — always
  • Pass 3: εtarget (online, for accurate CAS)
  • Pass 4: SKIP — use pre-computed danchor

Why v7 Works

Target: Online
Image-specific, accurate CAS detection
+
Anchor: Exemplar
Stable, real "clothed" from 16 diverse images
Online target (precise detection) + exemplar anchor (stable redirection) = best of both worlds. Plus 25% speedup.

Part 5

v10–v12: New Methods

Addressing the Safety–Quality Tradeoff

The Core Problem

v7's success comes at a cost: content destruction

v7 Hybrid Guidance

εfinal = εcfg − 15·M·(dtarget) + 15·M·(danchor)
Problem: dtarget contains both nudity AND legitimate content. Subtracting it with scale=15 destroys everything in the masked region.
Evidence: 42% of Ring-A-Bell outputs become "Not Relevant" in Qwen evaluation — the image content is destroyed, not just the nudity.

Ideal Solution

Remove only the nudity-aligned component from the prompt direction, preserve all other semantic content.
dsafe = dprompt − proj(dprompt → dtarget)
Nudity component (removed)   
Rest of prompt (preserved)

This is the key insight behind v10's proj_anchor mode.

v10 Prompt-Fidelity Projection Guidance

Surgical nudity removal via per-pixel projection

proj_anchor Mode

1. dot = (dprompt · dtarget) per pixel
2. proj_coeff = max(0, dot / ||dtarget||²)
3. dnudity = proj_coeff · dtarget (nudity component)
4. dsafe = dprompt − ts·M·dnudity
5. εsafe = ε∅ + cfg · dsafe
6. εfinal = (1−as·M)·εsafe + as·M·εanchor
Key: Only positive projection coefficients are used — we remove the nudity-aligned part, never accidentally amplify it.

Additional Modes

hybrid_fidelity: v7 hybrid + deviation clamping
Limit max change from εcfg to prevent content destruction
adaptive_cas: Scale guidance by CAS value
Higher CAS = more nudity = stronger guidance

v11 Stochastic Exemplar Ensemble

Same concept, diverse manifestations via controlled noise

Core Idea

Problem: Fixed z0 + "nudity" prompt → single deterministic trajectory → one anchor direction. But nudity manifests in many ways.
Solution: Generate K virtual exemplar directions by adding calibrated noise:
danchor,k = danchor + η · σd · √(1−ᾱt) · zk
Noise calibration:
  • η: user-controlled diversity (0 = deterministic)
  • σd: scale by direction's own std (in-distribution)
  • √(1−ᾱt): more noise at early steps, less at clean steps

Selection Strategy

best: Pick K-th anchor with highest CAS match to current image
weighted: CAS-weighted average of K anchors
average: Simple mean of all K (most robust)

Additional: eta-DDIM Sampling

--eta_ddim: Add controlled noise to the DDIM step itself.
xt−1 = predicted_mean + σeta·z — stochastic sampling from same model

v12 Cross-Attention WHERE + Projection HOW

Best spatial precision (v6) + best guidance (v10)

Spatial Mask Modes

crossattn: Pure attention-based mask
Probes UNet cross-attention layers for nudity token attention. Semantically precise.
noise_cas: v7-style noise-based spatial CAS
Per-pixel cosine similarity in noise space. More sensitive but noisier.
hybrid_mask: Weighted average of both
M = w·Mattn + (1−w)·Mnoise

Why Combine?

v6 alone:
Great WHERE (91% SR)
Poor safety (15%+ NudeNet)
+
v10 alone:
Great safety (6.65% NudeNet)
Poor quality (52% NotRel)
v12 goal: Combine precise WHERE (low NotRel) with surgical HOW (low NudeNet) to achieve both safety and quality.
Zero-cost WHERE: Cross-attention probing adds only one matmul per layer — no extra UNet calls.

Key Discovery: v3/dag — Area-Adaptive Guidance

DAG (area + magnitude adaptive) achieves best balance across all metrics

DAG Adaptive Formula

area_factor = 5 · (mask_area / total_pixels)
mag_scale = 1 + 4 · weight
correction = s · area_factor · mag_scale · M · dtarget
εsafe = εcfg − correction
Key insight: Guidance strength is proportional to detected area. Small nudity region → weak guidance. Large region → strong. This prevents over-correction on benign content.
Why v7 fails here: v7 uses fixed scale (15×) regardless of detected area, destroying content even when CAS barely triggers.

Key Insights & Lessons Learned

What Works

Area-adaptive scaling (v3/dag): guidance proportional to detected area prevents over-correction. Best overall balance.
Online target for detection: image-specific CAS is far superior to static directions (v8/v9 failure).
Cross-attention WHERE (v6): semantically precise masking yields highest SR (91%+) and best anchor alignment.

What Fails

Fixed-scale subtraction (v7 ts15+): removes nudity but destroys content. 42–75% NotRel.
Static exemplar target (v8/v9): too distant from per-image nudity. 42–76% unsafe.
Projection alone (v10/v11): low NudeNet but high NotRel (52%) — projection is too aggressive without area adaptation.

Next Directions

Combine DAG + Exemplar: v3's area-adaptive scaling with v7's exemplar anchor for even better results.
DAG + CrossAttn WHERE: v3's adaptive HOW with v6's semantic WHERE for precision.
Anchor alignment eval: Use anchor-friendly prompts to measure concept redirection quality.
