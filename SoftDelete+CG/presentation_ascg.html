Part 1

Safety-Guided Flow (SGF)

Kim, Kim & Park — ICLR 2026 Oral

SGF — Core Idea

Training-free negative guidance via MMD potential + CBF-proven critical time window

Key Insight: Use the gradient of the MMD (Maximum Mean Discrepancy) as a repulsive potential that pushes generated samples away from an unsafe reference dataset D−.
Three Contributions:
  • Unified framework: Shows SPELL and Safe Denoiser are special cases of MMD-gradient guidance
  • Critical time window: CBF analysis proves earlier guidance is strictly better
  • Plug-and-play: Compatible with SLD, SAFREE, and other base methods
ẋt = fθ(xt, t) + λ(t) ∇x E(xt)

Modified score function with MMD energy guidance

E(xt) ≡ MMD2kσ({xt}, D−)

MMD potential using RBF kernel

λ(t): Guidance schedule determined by CBF analysis — concentrate strength in early timesteps, decay to zero afterwards.

SGF — MMD Gradient as Repulsive Force

Closed-form gradient pushes xt away from nearest unsafe neighbors

∇xtMMD2 = 2/σ2Z [xt − ∑i wi(xt) yi]
wi(xt) = k(xt, yi) / (N · Z) — kernel-weighted attention
Closer unsafe samples exert stronger repulsion
Proposition 1 (Safe Denoiser): MMD gradient equals weighted kernel repellency up to a positive scalar.
Proposition 2 (SPELL): Gaussian force matches radial force via radius-bandwidth matching.

Unification Summary

MethodMechanismSGF View
SPELLRadial repulsion within radius rMMD with matched σ
Safe DenoiserWeighted kernel repulsionMMD with static σKDE
SGFMMD gradient + CBF windowUnified framework
Practical benefit: Both prior methods are recovered as special cases, enabling principled hyperparameter selection.

SGF — Theorem 2: Earlier Is Strictly Better

Control Barrier Function analysis proves optimal guidance scheduling

e∫L(τ)dτ h(x0) + μ · ĪL(sc) ≥ δ

Sufficient condition for safety (Theorem 2)

Setup:
  • h(x): barrier function — Safe = {h ≥ 0}, Unsafe = {h < 0}
  • ĪL: weighted guidance mass with decreasing weight w̄(u)

Key Result

Since w̄L(u) is strictly decreasing:

Under a fixed guidance budget ∫β(u)du,
shifting strength from later to earlier
strictly increases the safety bound.

Empirical validation (Fig. 4): Window [1.0, 0.8] (first 20% of steps) consistently achieves lowest ASR across all datasets.

ASCG — Classifier & Spatial Mask

4-Class Classifier

ClassIDRole
Benign0Ignore
Safe Clothed1Target
Harm Nude2Repel

gt = ∇z log p(y=1 | zt) − ∇z log p(y∈{2,3} | zt)

Simultaneously attract safe + repel harm

Spatial Mask Mt

M(p) = 1[ Φ((Hp − μ) / σ) ≥ τ(t) ]
GradCAM mode: Classifier gradient heatmap → z-score → CDF percentile → binary mask
Attention mode: Cross-attention maps for harmful tokens at 16×16, 32×32
Threshold schedule: Cosine annealing τstart → τend
Early: strict (preserve structure) → Later: broader (refine details)

Part 3

SGF + ASCG: Combination Experiments

Testing complementary strengths & finding what works

Complementary Strengths

SGF answers "when", ASCG answers "where" and "how"

SGF

  • When: CBF-proven time window
  • How: MMD repulsion (undirected)
  • Where: Global (entire latent)
  • Requires unsafe dataset D−
+

ASCG

  • When: CDF monitoring (heuristic)
  • How: Dual gradient (directed)
  • Where: GradCAM spatial mask
  • Requires trained classifier
Hypothesis:   Combine SGF's CBF window + MMD repulsion  +  ASCG's spatial mask + dual CG for best of both worlds.
Research question: Does MMD repulsion add value on top of directed classifier guidance? Does x0-space guidance improve over zt-space?

SGF + ASCG — Experiments

Three approaches tested: SGF x0-space + ASCG zt-space, schedule variants, ablations

# Hybrid pipeline (generate_std_sgf_ascg.py) for step, t in enumerate(timesteps): noise = unet(latents, t, prompt_embeds) # CFG if t_end ≤ t ≤ t_start: w = λ(t) # schedule weight # SGF: x0-space MMD repellency x0 = scheduler.step(noise,t,z).pred_original_sample x0 += w * sgf.repellency(x0) latents = add_noise(x0, ε, t) # DDIM step latents = scheduler.step(noise, t, latents) if t_end ≤ t ≤ t_start: # ASCG: z_{t-1}-space classifier guidance ascg_grad = guidance.compute_gradient( latents, t, harm=[2,3], thr=get_thr(step), scale=w*s_g) latents += ascg_grad

ASCG v1 → v2: Key Improvement

Post-DDIM guidance on zt-1 outperforms pre-DDIM guidance on zt

v1: Guidance on zt

  • ASCG modifies zt → then DDIM step
  • zt is noisier → classifier gradient less precise
  • Linear schedule λ(t) in [tend, tstart]
  • SR 82.3%, Full 3.8%, NR 13.9%

Tested but failed: SGF x0-space MMD

  • MMD repellency in x0-space hurts ASCG
  • σ(t) schedule worse than linear
  • Re-noising disrupts denoising trajectory

v2: Guidance on zt-1 (Post-DDIM)

  1. DDIM step first: zt → zt-1
  2. ASCG on zt-1: less noisy → sharper classifier gradient
  3. Linear schedule retained (empirically superior)
  4. SR 87.3%, Full 1.3%, NR 11.4%

Why zt-1 is better:
After the DDIM step, zt-1 is less noisy than zt. The classifier produces sharper, more accurate gradients on cleaner latents, leading to more precise guidance.
What did NOT help:
SGF MMD repellency (x0-space re-noising), σ(t) CBF schedule. Directed classifier guidance alone outperforms undirected MMD repulsion.

Key Findings & Takeaways

What worked

  • Post-DDIM ASCG: Applying guidance on zt-1 instead of zt → +5pp SR
  • Linear schedule: Outperforms CBF-derived σ(t) schedule
  • SGF time window concept: [1000, 400] range is effective
  • Spatial mask + dual gradient: Core ASCG components

What did NOT work

  • SGF MMD repellency: Hurts even at 1/30th scale — undirected repulsion conflicts with directed CG
  • x0-space re-noising: Injects noise that degrades trajectory
  • σ(t) schedule: Too aggressive early-weighting for spatial CG

Remaining Tasks

  1. Quality evaluation: COCO-30K FID / CLIP-Score for benign prompts
  2. Multi-attack: UnlearnDiff, MMA-Diffusion, P4D-Nudity
  3. Generalization: SD v2.1, SDXL, other concepts (violence, copyright)
  4. Reduce NR: 11.4% → target <5%
  5. ASCG + SAFREE: Test combining with text projection
Insight: Directed classifier guidance with spatial precision is more effective than undirected energy-based repulsion. The classifier already encodes richer semantics than kernel distance from reference images.
