Part 1

ASCG — Adaptive Spatial Classifier Guidance

Motivation

Problem

  • T2I diffusion models generate harmful content
  • Fine-tuning methods permanently alter weights
  • Existing inference-time methods lack spatial awareness

Our Goal

  • Training-free — no weight modification
  • Adaptive — detect when to intervene
  • Spatially precise — target only harmful regions
  • Quality-preserving — safe areas untouched
Key Insight: Harmful content is spatially localized. By targeting only the harmful regions at the right timesteps, we erase concepts while preserving quality.

WHEN & WHERE — CDF-based Detection

WHEN: Global Monitoring

P̄(harm | zt) = Φ( (μheatmap − μtrain) / σtrain )
  1. GradCAM heatmap H(zt, charm)
  2. Z-score against training statistics
  3. Gaussian CDF → P(harm) ∈ [0,1]
  4. If P̄ > τ̄ → activate guidance

WHERE: Spatial Mask

M(p) = 𝟙[ Φ( (Hp − μtopk) / σtopk ) ≥ τspatial ]
  • Per-pixel CDF normalization
  • Binary mask: harmful=1, safe=0
  • Cosine threshold schedule:
    τstart=0.5 → τend=0.1
    Strict early (structure) → Broad late (details)

HOW — Bidirectional Gradient Injection

zt' = zt + λ · w(t) · M(zt) ⊙ ( ∇safe − α · ∇harm )
∇safe = ∇zt log pφ(csafe | zt)
Pull toward safe_clothed (class 1)
∇harm = ∇zt log pφ(charm | zt)
Push from nude + color classes (2,3)

ASCG Algorithm

Algorithm 1   Denoising step of Latent DDIM incorporating ASCG
Input: Latent zt, ratio αt, denoiser εθ, prompt p, latent classifier pφ
Output: Latent zt−1 at time t − 1
1:   P̄t,c , Pt,c ← ComputeSaliency(zt ; pφ) ▷ Equations 2-4
2:   if P̄t,c > τ̄ then ▷ WHEN to guide
3:   mt ← ComputeMask(Pt,c) ▷ WHERE to guide (Eq. 5)
4:   gt ← ∇zt log pφ(csafe | zt) − λrep · ∇zt log pφ(charm | zt) ▷ HOW (Eq. 6)
5:   gtspatial ← ComputeSpatial(gt , mt) ▷ Equation 7
6:   ε̂t ← εθ(zt, t, p) − √(1 − αt) · gtspatial ▷ Equation 8
7:   else
8:   ε̂t ← εθ(zt, t, p)
9:   end if
10: zt−1 ← DDIM(zt, αt, ε̂t) ▷ Equation 1
Guidance is injected into the noise estimate ε̂t (line 6), not directly into zt. Standard DDIM step follows (line 10).

Part 2

Current Limitation of ASCG

"When" monitoring is the performance bottleneck

The "When" Problem

WHERE and HOW work well, but WHEN holds back performance

WHERE ✓

GradCAM spatial mask accurately localizes harmful regions

HOW ✓

Bidirectional gradient effectively pushes toward safety

WHEN ✗

CDF monitoring has limited discriminability in noisy latent space

Evidence from experiments: Even with optimal WHERE/HOW parameters, ASCG on vanilla SD v1.4 achieves only ~49% SR. The monitoring often fails to detect harmful content forming in early-to-mid timesteps, or triggers guidance at the wrong moments.

Current Monitoring Approaches

Multiple "When" algorithms explored, none fully satisfactory

ModeMechanismProsCons
GradCAM CDF P(harm) via heatmap z-score per step Per-step adaptivity Noisy at high t, threshold-sensitive
Z0-Trigger Predict x̂0, classify at t=0 Cleaner signal (denoised) x̂0 prediction noisy at early steps
Z0-Sticky Once triggered, guide all remaining No missed steps Over-guides, quality degradation
Dual Z0-sticky + GradCAM refinement Combined signal Complex, many hyperparameters

Research Question

Can we find a better "When" signal that reliably identifies the critical timesteps where harmful concepts crystallize in the denoising trajectory?

SDErasure (ICLR 2026)

A promising "When" signal: SSScore

SDErasure — Overview

Concept erasure via adaptive denoising trajectory shifting

Core idea: Instead of erasing concept everywhere, identify critical timesteps where the model distinguishes target from anchor, and shift trajectory only there.

* SDErasure is a fine-tuning method. We are interested in borrowing its SSScore for our inference-time framework.

SSScore — Step Separability Score

Quantifies per-timestep concept discriminability

St = exp(−Lc) / ( exp(−Lc) + exp(−La) ) = 1 / (1 + exp(Lc − La))
Lc = ||εθ(xt, ctarget, t) − ε||²
La = ||εθ(xt, canchor, t) − ε||²
Interpretation:
  • St ≈ 1.0 → Model recognizes target at step t
  • St ≈ 0.5 → No discrimination → skip
  • St ≈ 0.0 → Anchor dominates

Why SSScore for ASCG?

SSScore tells us exactly which timesteps the model "decides" to generate harmful content. This is precisely the "When" signal ASCG needs!

Caveat: Requires semantic anchor. Empty anchor → St ≈ 0.5 everywhere (no signal)

Integration Approaches

A. SSScore-only Monitoring

Precompute SSScore schedule → guide only at St > threshold
Pro: Focused on critical timesteps
Con: Schedule from training data, may not generalize to all prompts

B. SSScore + CDF Hybrid

SSScore for coarse schedule + CDF for per-image refinement
Pro: Best of both worlds
Con: More computation

C. SDErasure + ASCG Stack

Fine-tuned UNet (v12) + ASCG as safety net
Pro: 92.4% SR base + catch remaining failures
Con: Requires fine-tuning (not purely inference-time)

D. SSScore-weighted Guidance

λ(t) = λbase · St (continuous weighting)
Pro: Smooth, no hard threshold
Con: Anchor dependency
