# Safe Latent Diffusion (SLD) - 상세 분석

## 기본 정보
- **논문 제목**: Safe Latent Diffusion: Mitigating Inappropriate Degeneration in Diffusion Models
- **저자**: Patrick Schramowski*, Manuel Brack*, Bjorn Deiseroth, Kristian Kersting (*equal contribution)
- **소속**: DFKI, Hessian.AI, TU Darmstadt, Aleph Alpha, LAION
- **학회/년도**: CVPR 2023 (arXiv: 2211.05105)
- **코드**: HuggingFace diffusers 파이프라인으로 공식 통합 (`stable_diffusion_safe`)

---

## 핵심 문제 정의

Text-to-Image diffusion model (특히 Stable Diffusion)이 대규모 비정제 데이터셋(LAION-5B)으로 학습되면서, 명시적으로 unsafe한 프롬프트뿐 아니라 **겉보기에 무해한 프롬프트에서도** 부적절한 이미지(nudity, violence, hate 등)를 생성하는 문제.

추가로, 학습 데이터의 편향이 모델에 전이되어 **인종/민족 편향**(예: "japanese body" 프롬프트에서 90% 이상 누드 생성)을 강화하는 문제를 다룸.

핵심 질문: **추가 학습 없이, 모델 자체가 학습한 "적절함/부적절함" 지식을 활용하여 inference 단계에서 unsafe content를 억제할 수 있는가?**

---

## 핵심 방법론

### 1. 기본 아이디어: Safety Guidance via Classifier-Free Guidance 확장

표준 CFG는 2개의 noise prediction을 사용:
- unconditional: `epsilon_theta(z_t)` (null prompt)
- conditional: `epsilon_theta(z_t, c_p)` (user prompt)

SLD는 **3번째 noise prediction**을 추가:
- safety concept conditioned: `epsilon_theta(z_t, c_S)` (unsafe concept prompt)

이를 통해 prompt 방향으로 가이드하면서 **동시에** unsafe concept 방향에서 멀어지도록 유도.

### 2. 핵심 수식

#### 기본 CFG (Eq. 2)
```
epsilon_tilde(z_t, c_p) = epsilon_theta(z_t) + s_g * (epsilon_theta(z_t, c_p) - epsilon_theta(z_t))
```

#### SLD의 확장된 score estimate (Eq. 3)
```
epsilon_tilde(z_t, c_p, c_S) = epsilon_theta(z_t) + s_g * (epsilon_theta(z_t, c_p) - epsilon_theta(z_t) - gamma(z_t, c_p, c_S))
```

여기서 `gamma`가 safety guidance term.

#### Safety Guidance Term (Eq. 4)
```
gamma(z_t, c_p, c_S) = mu(c_p, c_S; s_S, lambda) * (epsilon_theta(z_t, c_S) - epsilon_theta(z_t))
```

`mu`는 element-wise scaling factor, `s_S`는 safety guidance scale.

#### Element-wise Scaling Factor mu (Eq. 5-6)
```
mu(c_p, c_S; s_S, lambda) = 
  max(1, |phi|),  where epsilon_theta(z_t, c_p) - epsilon_theta(z_t, c_S) < lambda
  0,              otherwise

phi = s_S * (epsilon_theta(z_t, c_p) - epsilon_theta(z_t, c_S))
```

**핵심 메커니즘**: prompt conditioned estimate과 safety concept estimate의 차이가 threshold `lambda` 미만인 latent 차원들만 선택적으로 개입. 즉, **prompt가 unsafe concept과 유사한 방향으로 가고 있는 차원만** safety guidance를 적용.

#### Warm-up Period delta (시간적 제어)
```
gamma(z_t, c_p, c_S) := 0  if t < delta
```
처음 `delta` 스텝 동안은 safety guidance를 적용하지 않아 이미지의 전체적 구도를 먼저 형성. 이후 세부 사항 단계에서 unsafe 요소를 제거.

#### Momentum (Eq. 7-8)
```
gamma_t = mu * (n_s - n_empty) + s_m * nu_t
nu_{t+1} = beta_m * nu_t + (1 - beta_m) * gamma_t
```
- `s_m`: momentum scale (0~0.5)
- `beta_m`: momentum decay (0~0.7)
- 연속적으로 같은 방향으로 가이드되는 차원에 대해 가속, 강한 unsafe 영역을 더 적극적으로 제거

### 3. Unsafe Concept 정의

텍스트 기반으로 unsafe concept `S`를 정의:
> *"hate, harassment, violence, suffering, humiliation, harm, suicide, sexual, nudity, bodily fluids, blood, obscene gestures, illegal activity, drug use, theft, vandalism, weapons, child abuse, brutality, cruelty"*

이 텍스트를 text encoder에 넣어 `c_S`를 생성. 사용자가 자유롭게 수정/확장 가능.

### 4. 4가지 강도 설정 (Configuration Sets)

| Config | delta (warm-up) | s_S (safety scale) | lambda (threshold) | s_m (momentum scale) | beta_m (momentum decay) |
|--------|-------|------|--------|------|--------|
| **Hyp-Weak** | 15 | 200 | 0.0 | 0.0 | - |
| **Hyp-Medium** | 10 | 1000 | 0.01 | 0.3 | 0.4 |
| **Hyp-Strong** | 7 | 2000 | 0.025 | 0.5 | 0.7 |
| **Hyp-Max** | 0 | 5000 | 1.0 | 0.5 | 0.7 |

- Weak: 표면적 blood splatter 정도 제거
- Medium: 비키니 수준까지 nudity 억제
- Strong: 옷을 입힌 수준으로 변환
- Max: 거의 모든 inappropriate content 제거 (FP 증가 가능)

## Training-Free 여부

**완전히 Training-Free.** 모델 가중치 수정 없음, 추가 학습 없음, 외부 classifier 불필요. Inference 시 3번의 UNet forward pass만 필요 (기존 CFG의 2번 대비 1번 추가). 모델이 pre-training에서 학습한 "적절함" 지식을 그대로 활용.

## WHEN / WHERE / HOW 프레임워크 분석

### WHEN: 언제 guidance를 적용하는가?

- **Warm-up 이후 모든 timestep에 적용** (`t >= delta`)
- Warm-up period `delta` (0~20 steps): 처음 delta 스텝 동안은 safety guidance 없이 이미지 구도 형성
- Warm-up 동안에도 momentum은 축적됨 (이후 적용 시 가속 효과)
- **조건부 적용 아님**: CAS 같은 semantic 탐지 메커니즘 없이, 모든 프롬프트에 대해 항상 적용
- 즉, **benign 프롬프트에도 항상 safety guidance가 켜져 있음** -> FP 가능성 내재

### WHERE: 어디에 적용하는가?

- **Latent space에서 element-wise 선택적 적용**
- mu 함수가 핵심: `epsilon_theta(z_t, c_p) - epsilon_theta(z_t, c_S) < lambda`인 차원만 선택
- 이는 prompt prediction이 safety concept prediction과 가까운 latent 차원을 식별
- **Spatial mask가 아닌 latent-dimension-wise mask** (64x64 latent의 각 element를 독립적으로 판단)
- Cross-attention map 기반이 아니라, noise prediction 값의 차이를 직접 비교
- 결과적으로 "unsafe한 부분"에 해당하는 latent 차원만 수정하는 semi-spatial 접근

### HOW: 어떻게 개입하는가?

- **Noise prediction 수정**: CFG의 score estimate에 safety guidance term `gamma`를 빼줌
- 직관: unconditional -> prompt 방향으로 가이드하되, unsafe concept 방향 성분을 빼줌
- `epsilon_safe = epsilon_cfg - gamma` 형태
- Momentum으로 연속적인 unsafe 차원에 대해 가속 적용
- x-prediction 값을 [-1, 1]로 clipping하여 latent 파괴 방지
- Attention 수정 아님, noise space에서의 직접적 조작

## 추가 기여: I2P Benchmark

- **Inappropriate Image Prompts (I2P)**: 4,703개의 real-world unsafe 프롬프트 데이터셋
- 7개 카테고리: hate, harassment, violence, self-harm, sexual, shocking, illegal activity
- Lexica.art에서 수집한 실제 사용자 프롬프트
- Q16 classifier + NudeNet으로 자동 평가
- 우리 프로젝트에서 사용하는 I2P subset의 원본 소스

## 장점

1. **완전한 Training-Free**: 모델 가중치 수정 불필요, inference-only 수정
2. **유연한 강도 조절**: 4단계 사전 설정 + 연속적 하이퍼파라미터 조절 가능
3. **텍스트 기반 concept 정의**: unsafe concept을 자연어로 정의, 확장 용이
4. **HuggingFace 공식 통합**: 실용적 배포, diffusers 파이프라인으로 바로 사용 가능
5. **Multi-concept 지원**: 하나의 safety prompt에 여러 unsafe concept 나열 가능
6. **I2P 벤치마크 제공**: 후속 연구의 표준 평가 도구가 됨
7. **Latent-level selective 적용**: mu 함수로 불필요한 개입 최소화 시도

## 한계

1. **모든 프롬프트에 항상 적용 (No WHEN detection)**:
   - Benign 프롬프트에도 safety guidance가 적용됨
   - FP 위험: "beach scene", "art model" 등 합법적 프롬프트의 이미지 변형 가능
   - 이미지 구도/스타일이 미묘하게 바뀔 수 있음

2. **Spatial precision 부족 (Coarse WHERE)**:
   - Latent dimension-wise mask이지 semantic spatial mask가 아님
   - 이미지의 어느 영역이 unsafe인지 명시적으로 파악하지 못함
   - Cross-attention 기반 공간적 추론 없음

3. **Safety concept의 일반성 한계**:
   - 하나의 general safety prompt로 7개 카테고리를 모두 커버
   - Concept별 최적화가 안 됨 -> shocking (52% -> 13%) vs sexual (35% -> 6%)로 효과 차이 큼
   - "what is unsafe"의 판단이 텍스트 프롬프트 기반이라 세밀한 구분 어려움

4. **Computational overhead**:
   - 3번의 UNet forward pass 필요 (기존 2번 대비 50% 증가)
   - Momentum 계산 추가

5. **Threshold 민감도**:
   - lambda, s_S, delta 등 다수의 하이퍼파라미터
   - 최적 설정이 concept/use case마다 다름
   - 사용자가 4개 preset 중 선택해야 하는 주관적 판단 필요

6. **Adversarial robustness 미검증**:
   - Ring-A-Bell, MMA 같은 adversarial prompt에 대한 평가 없음
   - 교묘하게 unsafe concept을 우회하는 프롬프트에 취약할 가능성

7. **완전한 제거 불가**:
   - Hyp-Max에서도 9%의 inappropriate image 잔존
   - Q16 classifier의 FP를 고려해도 일부 unsafe content 통과

## 우리 방법 (CAS + Spatial CFG)과의 관계/차이점

### 공통점
- 둘 다 **training-free** inference-time 개입
- 둘 다 **CFG 확장** 기반: noise prediction을 수정하여 unsafe 방향에서 벗어남
- 핵심 수식 구조 유사: `epsilon_safe = epsilon_cfg - s * factor * (epsilon_target - epsilon_null)`
- 둘 다 **추가 UNet forward pass** 필요 (unsafe concept에 대한 noise prediction)
- 모델 가중치 동결, 외부 classifier 불필요

### 핵심 차이점

| 측면 | SLD | CAS + Spatial CFG (Ours) |
|------|-----|--------------------------|
| **WHEN** | 모든 프롬프트에 항상 적용 (warm-up 이후) | CAS score로 **조건부 적용** - cos(d_prompt, d_target) > threshold일 때만 |
| **WHERE** | Latent element-wise mask (mu 함수) | Cross-attention 기반 **spatial mask** - 이미지의 어느 영역이 unsafe인지 명시적 파악 |
| **HOW** | 일반적 safety prompt 기반 | **Exemplar-based concept direction** - 구체적 nudity exemplar로 정밀한 concept 방향 계산 |
| **Concept 정의** | 텍스트 프롬프트 (general) | CLIP exemplar embedding 기반 (specific) |
| **FP 제어** | 강도 preset으로 간접 제어 | CAS threshold로 직접 제어 (benign은 아예 건너뜀) |
| **Spatial 정밀도** | 없음 (전체 latent) | 있음 (attention map 기반 body part 영역만) |

### SLD의 기여를 우리가 활용하는 부분
1. **기본 수식 프레임워크**: SLD의 Eq. 3이 우리 HOW의 출발점
2. **I2P 벤치마크**: 우리 실험의 I2P subset이 SLD 논문에서 제안된 것
3. **Momentum 아이디어**: 연속적 guidance 가속의 참고 가능
4. **Warm-up 개념**: timestep별 guidance 강도 조절의 선례

### 우리 방법의 SLD 대비 개선점
1. **WHEN (CAS)**: Benign 프롬프트에 개입하지 않아 **FP 대폭 감소** + 이미지 품질 보존
2. **WHERE (Spatial)**: 이미지 전체가 아닌 **unsafe 영역만** 수정하여 나머지 보존
3. **Concept Direction**: General safety text 대신 **exemplar 기반** 정밀한 unsafe 방향 계산
4. **Adaptive Guidance**: 프롬프트의 unsafety 정도에 따라 guidance 강도가 자동 조절

### SLD가 우리보다 나은 점
1. **Multi-concept 통합**: 하나의 safety prompt로 7개 concept 동시 대응 (우리는 concept별 anchor 필요)
2. **구현 단순성**: 하이퍼파라미터 4개, 코드 수십 줄로 구현 가능
3. **HuggingFace 생태계 통합**: 실용적 배포 완료
4. **광범위한 평가**: I2P 전체 7개 카테고리 + ethnic bias + user study + DrawBench

## 인용 시 핵심 포인트

SLD는 training-free concept erasing의 **선구적 연구**로, CFG를 확장하여 safety guidance를 추가하는 기본 프레임워크를 확립했다. 그러나 WHEN (모든 프롬프트에 항상 적용)과 WHERE (spatial precision 없음)에서의 한계가 있으며, 우리 방법은 이 두 축을 CAS와 Spatial mask로 보완하여 **선택적이고 정밀한** concept erasing을 달성한다.

우리 논문에서의 포지셔닝: *"SLD는 HOW (CFG 확장으로 unsafe 방향에서 벗어남)를 확립했으나, WHEN과 WHERE를 다루지 않아 benign content에 대한 불필요한 개입과 spatial 정밀도 부족이라는 한계가 있다. 우리는 example-based WHEN (CAS) + WHERE (Spatial attention mask)를 추가하여 이 gap을 메운다."*
