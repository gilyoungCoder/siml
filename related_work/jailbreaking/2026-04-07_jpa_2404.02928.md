# Jailbreaking Prompt Attack: A Controllable Adversarial Attack against Diffusion Models (arXiv:2404.02928)

- Title: **Jailbreaking Prompt Attack: A Controllable Adversarial Attack against Diffusion Models**
- Authors: Jiachen Ma, Yijiang Li, Zhiqing Xiao, Anda Cao, Jie Zhang, Chao Ye, Junbo Zhao
- Date checked: 2026-04-07
- Primary sources:
  - arXiv abstract: https://arxiv.org/abs/2404.02928
  - arXiv PDF: https://arxiv.org/pdf/2404.02928.pdf

---

## 1. 한 줄 요약
이 논문은 **text embedding space에 이미 NSFW concept direction이 존재한다**는 관찰을 바탕으로, 그 방향을 포착한 뒤 **discrete prefix prompt optimization** 으로 safety checker를 우회하는 **black-box prompt attack** 을 제안한다. 핵심은 evaluator-feedback loop보다는, **embedding-guided prompt construction** 쪽에 더 가깝다.

## 2. 문제의식
저자들은 기존 공격의 문제를 두 가지로 본다.

1. 많은 white-box 공격은 diffusion model 내부 접근이 필요해 온라인 서비스에 쓰기 어렵다.
2. 일부 black-box 공격은 느리거나, NSFW 이미지는 만들어도 **원래 prompt semantics와 일치성이 떨어진다**.

즉, 논문의 타깃은
- black-box 가능
- semantic fidelity 유지
- 빠른 자동화
세 가지를 동시에 만족하는 공격이다.

## 3. 핵심 아이디어
### 3.1 NSFW concept는 embedding space에 이미 있다
논문은 unsafe concept가 단순한 금지어 리스트 수준이 아니라,
**고차원 text embedding 공간 안의 특정 방향(direction)** 으로 존재한다고 본다.

예를 들어 “nudity” 같은 개념을 직접 쓰지 않고도,
그 개념과 관련된 embedding 방향을 원래 prompt embedding에 더하면,
이미지 생성이 unsafe 방향으로 기울 수 있다는 것이다.

### 3.2 antonym pair로 concept direction을 만든다
JPA는 ChatGPT가 만든 antonym pair를 이용한다.
예:
- nude vs clothed

이런 쌍들의 embedding 차이를 평균내서,
특정 NSFW concept를 대표하는 벡터 `r` 을 만든다.
그 다음 원래 prompt embedding에 `λ · r` 을 더해서,
원래 의미는 유지하면서도 unsafe concept가 실리도록 한다.

### 3.3 최종 출력은 “prefix prompt”
연속 공간에서 만든 target embedding을 그대로 쓸 수는 없으니,
논문은 discrete text space에서 **prefix prompt** 를 최적화한다.
즉 원래 prompt 앞에 learnable token들을 붙이고,
그 prefix가 목표 embedding과 align되도록 최적화한다.

이때 중요한 기술이 두 가지다.
- **soft assignment**: discrete token search를 연속 relaxation으로 다룸
- **gradient masking**: 민감 단어 자체가 선택되지 않도록 gradient를 막음

결국 결과물은 대놓고 금지어가 들어간 prompt가 아니라,
표면적으로는 비교적 무해해 보이지만 embedding상 unsafe concept를 끌어오는 prefix다.

---

## 4. 이 논문이 “evaluator loop” 계열인가?
부분적으로는 맞지만, 중심은 다르다.

### 맞는 부분
- 공격 성공 여부를 detector로 평가한다.
- ASR, FID, CLIPScore 등으로 성능을 본다.
- safety checker 우회 여부를 중요한 목표로 둔다.

### 하지만 핵심은 evaluator-driven iterative refinement가 아님
RPG-RT처럼
- 시스템 피드백을 선호 데이터로 바꾸고
- attacker를 계속 fine-tune하는 구조는 아니다.

JPA는 더 가깝게 말하면:
- **embedding space에서 target concept를 잡고**
- **prefix prompt를 discrete optimization으로 찾는 방식**
이다.

즉 사용자의 관찰대로,
이 논문은 “feedback을 받아 계속 prompt를 깎는 구조”보다는,
**한 번의 잘 설계된 optimization procedure로 강한 adversarial prefix를 찾는 구조**에 더 가깝다.

## 5. 방법 디테일
### 5.1 공격 입력
- 원래 민감 프롬프트 `pt`
- 목표 NSFW concept에 대한 contrastive description `<r+, r->`
  - 예: `<nudity, clothed>`

### 5.2 concept embedding 생성
- 여러 antonym pair를 ChatGPT로 생성
- 각 pair의 embedding 차이를 평균해 `r` 생성
- `T(pr) = T(pt) + λ·r`

여기서 `λ`는 컨트롤 노브다.
논문이 특히 강조하는 부분 중 하나가,
**unsafe concept rendering strength를 조절할 수 있다**는 점이다.
즉 완전한 full nudity만이 아니라,
약한 수준에서 강한 수준까지 concept emergence를 어느 정도 제어할 수 있다고 주장한다.

### 5.3 discrete optimization
- prefix 길이 `k`를 둠
- 각 위치는 vocabulary 전체에 대한 soft assignment로 표현
- gradient ascent로 target embedding과 alignment를 최대화
- 최종적으로 argmax token 선택

### 5.4 gradient masking
민감 단어를 직접 쓰면 text-based safety checker에 걸릴 가능성이 높으므로,
금지어 리스트에 속한 단어는 gradient를 강하게 마스킹해 선택 가능성을 낮춘다.

이 점은 꽤 중요하다.
즉 JPA는 “unsafe semantics를 강하게 넣자”와 “검열 단어는 피하자”를 동시에 최적화한다.

## 6. 핵심 결과
### 6.1 속도
논문 Table 3 기준, prompt당 공격 시간은:
- P4D: 30.70 min
- UnlearnDiff: 26.29 min
- random: 29.08 min
- QF-Attack: 55.42 min
- SneakyPrompt: 59.18 min
- Ring-A-Bell: 62.50 min
- **JPA: 6.72 min**

이건 논문의 강한 포인트다.
즉 JPA는 단지 성공률만이 아니라, **자동화 가능성과 시간 효율**을 강하게 민다.

### 6.2 온라인 서비스
Table 12 텍스트 추출 기준에서,
JPA는 최소한 다음을 보여준다.
- DALL·E 2, Stability AI 같은 온라인 서비스에도 적용 가능
- online setting에서 ASR/CLIPScore/time을 보고
- fully automated attack framework를 주장함

텍스트 추출 품질상 online 표는 완전히 깨끗하진 않았지만,
본문 설명상 저자들은 기존 방법 대비
- black-box applicability
- semantic alignment
- faster execution
을 핵심 실적으로 제시한다.

### 6.3 semantic fidelity
논문은 기존 공격의 가장 큰 약점 중 하나를
“NSFW 이미지는 나오지만 원래 prompt와 의미적으로 동떨어진다”로 본다.
JPA는 이를 줄이는 게 핵심 기여다.
즉 단순한 우회가 아니라,
**원래 장면/서술과 unsafe concept를 함께 유지하려는 공격**이다.

## 7. 강점
### 7.1 매우 명쾌한 공격 surface
- 내부 diffusion gradient 불필요
- target model 없어도 됨
- CLIP-like embedding 구조 활용

실전에서 쓸 수 있는 black-box 공격으로서 구조가 명확하다.

### 7.2 “semantic fidelity”를 전면에 둠
기존 prompt jailbreak는 종종 “뚫리기만 하면 됨”인데,
JPA는 원래 prompt 의미를 유지하면서 unsafe concept를 주입하려 한다.
이건 red-teaming 관점에서 더 위험하고, 동시에 더 현실적이다.

### 7.3 controllability
`λ`를 통해 concept rendering 강도를 어느 정도 조절할 수 있다는 주장은,
공격이 단지 binary success/failure가 아니라
**continuous manipulation** 으로도 이해될 수 있음을 보여준다.

---

## 8. 약점 / 비판 포인트
### 8.1 evaluator-loop 적응성은 약함
RPG-RT와 비교하면,
JPA는 target system의 feedback을 쌓아 가며 adaptive하게 학습하는 구조가 아니다.
따라서 완전히 새로운 상용 방어 조합이 등장했을 때,
**실시간 적응성은 RPG-RT보다 약할 가능성**이 있다.

### 8.2 CLIP-space 가정 의존성
이 공격은 본질적으로
“unsafe concept가 CLIP-style text embedding에서 안정적으로 방향성을 갖는다”는 가정에 의존한다.
만약 target system의 text encoding / alignment 구조가 많이 다르면,
transfer strength가 줄 수 있다.

### 8.3 prompt-space 한계
아무리 잘해도 결국 prompt-level perturbation이다.
즉 unlearning처럼 내부 모델 파라미터/latent dynamics를 바꾼 방어에는,
latent-space attack만큼 직접적이지 않을 수 있다.

## 9. 우리 연구 맥락에서의 해석
이 논문은 사용자의 직관 중 절반을 지지한다.

- 맞는 점: diffusion jailbreak에서 여전히 **prompt optimization이 핵심 전장**이다.
- 덜 맞는 점: 이 논문은 evaluator feedback loop보다는 **embedding-guided optimization** 이 중심이다.

즉,
- “대부분 prompt를 깎는다” → 맞음
- “대부분 evaluator를 두고 iterative feedback으로 깎는다” → JPA 자체는 그쪽보단 약함

그래서 세 논문 중 분류하면:
- **RPG-RT**: evaluator/feedback loop 대표
- **JPA**: embedding-guided prompt optimization 대표
- **IVO**: latent optimization 대표

---

## 10. 다른 두 논문과의 관계
### vs RPG-RT
- JPA는 static optimization 색이 강함
- RPG-RT는 interaction-driven adaptive optimization 색이 강함
- JPA는 “좋은 prefix를 찾자”
- RPG-RT는 “target feedback을 배우는 attacker를 만들자”

### vs IVO
- JPA는 prompt를 건드림
- IVO는 initial latent를 건드림
- JPA는 일반 T2I safety checker / concept removal 우회
- IVO는 특히 **unlearned DM의 dormant memory 재활성화**에 초점

---

## 11. 최종 메모
JPA는 현재도 여전히 중요한 기준점이다.
왜냐하면 이 논문이 보여준 건,
**굳이 diffusion guidance 자체를 직접 건드리지 않아도**,
text embedding space와 discrete prefix optimization만으로도 꽤 강한 jailbreak가 가능하다는 점이기 때문이다.

즉 사용자의 관찰처럼,
실전적 jailbreak의 중심축은 여전히 **prompt-side optimization** 이고,
JPA는 그 계열의 대표적인 논문으로 보는 게 맞다.
