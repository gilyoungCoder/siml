공통 프레임워크: WHEN · WHERE · HOW

모든 버전이 이 3단계를 공유. 버전마다 각 단계의 구현이 다름.

WHEN — 언제 가이드?

CAS = cos(dprompt, dtarget) > 0.6
  • 프롬프트가 nudity와 얼마나 align되는지
  • Sticky: 한번 trigger되면 유지
  • COCO FP = 5% (안전 프롬프트 거의 안 걸림)

WHERE — 어디를 가이드?

  • Noise-based: 3×3 per-pixel CAS (v3~v5, v7, v10, v11)
  • CrossAttn: attention probe로 nudity token 위치 (v6, v12)
  • Sigmoid + Blur → soft mask

HOW — 어떻게 가이드?

  • SLD: ε − s·M·dtarget
  • DAG: area-adaptive scaling
  • Hybrid: subtract target + add anchor
  • Projection: nudity 성분만 제거
평가 지표: SR (Qwen) — 메인 NudeNet — safety Falcon NSFW SigLIP 5-class VQAScore — fidelity Anchor VQA — redirection

핵심 인사이트

왜 v3/dag가 최강인가?

Area-adaptive scaling: guidance 강도가 detected area에 비례.
  • 작은 nudity 영역 → 약한 guidance → 비-nudity 보존
  • 큰 nudity 영역 → 강한 guidance → 확실히 제거
v7의 문제: 고정 scale 15×로 무조건 강하게 → nudity가 아닌 배경/사람/맥락까지 전부 파괴 → NotRel 42%+
v10/v11의 문제: Projection은 nudity 성분만 제거하지만, spatial mask가 너무 넓어서 결국 NotRel 52%

다음 방향

1. DAG + Exemplar: v3의 area-adaptive + v7의 exemplar anchor → UNet 3 call + 최고 안전성
2. DAG + CrossAttn WHERE: v3의 adaptive HOW + v6의 정밀 WHERE → 정밀 위치 + 적응적 강도
3. 데이터셋 확장: MMA, P4DN, UnlearnDiff에서도 v3/dag가 최강인지 검증
4. SAFREE 비교: State-of-the-art baseline과 head-to-head
