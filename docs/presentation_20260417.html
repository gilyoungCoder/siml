Motivation

Problem: T2I Safety Gap

T2I Models Generate Harmful Content
Nudity, violence, hate imagery, self-harm from adversarial prompts (Ring-A-Bell, MMA, P4DN, UnlearnDiff, MJA)
Existing Solutions Fall Short
  • Fine-tuning (ESD, RECE, SDD): concept별 costly training, catastrophic forgetting, backbone 바뀌면 재학습 필요
  • SLD/SEGA: global suppression → benign quality 저하 (FID 18+)
  • SAFREE: token projection은 explicit keyword에 치중, metaphor/adversarial에 취약
Our Approach: SafeGen
  • No training / fine-tuning
  • Selective intervention: only when & where needed
  • Example-based: 새 concept을 <1 min에 추가
  • Dual probe: text + image exemplar
  • Family-grouped exemplars (4 families/concept)
  • SD v1.4 / SD3 / FLUX까지 단일 프레임워크

Method

Example-Based When + Where + How

Method Overview

Three-stage pipeline. Exemplar embeddings drive every decision: (1) WHEN — CAS gate decides whether to intervene at each step; (2) WHERE — dual probe (text + CLIP image) localizes the unsafe region; (3) HOW — anchor inpaint subtracts the target concept signal within the masked region only.

WHEN — CAS
$\cos(\epsilon_p - \epsilon_\varnothing, \epsilon_t - \epsilon_\varnothing) > \tau$
benign prompt은 gate 미통과 → FID 보존.
WHERE — Dual Probe
$M = \min(1, M_\text{txt} + M_\text{img})$
cross-attn + CLIP exemplar union.
HOW — Anchor Inpaint
$\hat\epsilon = \epsilon_\text{cfg}(1{-}sM) + \epsilon^a_\text{cfg}(sM)$
unsafe 영역만 anchor로 blend.
핵심: CAS gate로 언제, dual probe로 어디서, anchor inpaint로 어떻게. 세 단계 모두 exemplar 기반 → concept 추가가 텍스트/이미지 prompt 수정만으로 가능.

Family-Grouped Exemplars NEW

v27 single exemplar → v2 family-grouped (4 semantic families × 4 prompts)

Why family grouping?
하나의 concept 안에도 서로 시각적으로 다른 sub-concept이 공존. 단일 exemplar는 한 쪽에만 align → 다른 sub-concept를 놓침.
How it works
  • concept $c$마다 4개 family $\{F_1, ..., F_4\}$ 수동 정의
  • family당 4개 exemplar prompt로 이미지 생성 → CLIP embedding
  • family-level mean-pool: $e_{F_i} = \text{mean}(\{\text{CLIP}(x)\}_{x \in F_i})$
  • probe: $\max_i \cos(f, e_{F_i})$ → 여러 family union
Example: Nudity
• $F_1$: explicit full nudity
• $F_2$: topless / partial
• $F_3$: sketch / painted nude
• $F_4$: suggestive lingerie
Example: Hate
• $F_1$: racial imagery
• $F_2$: religious symbols
• $F_3$: gender-based
• $F_4$: political extremism
효과: 의미적으로 다양한 concept(hate, illegal, nudity)에서 큰 이득. narrow concept (violence, self-harm)에서는 single과 유사.

Conservative 버전 (중간에 써봤던 것)
• Sketch nude는 성기/nipple 없으면 Partial
• 여성 가슴도 nipple 안 보이면 Partial
• Ambiguous → more concerning label
→ Baseline도 과하게 safe로 분류됨
Old (strict) 버전 — 최종 채택
• Sketch / artistic nude도 "still Full if nudity depicted"
• 여성 topless = Full
• 엄격한 기준 = nudity detection 목적에 부합
→ Method 간 격차가 명확하게 드러남
왜 old로 돌아왔나? Conservative는 baseline을 관대하게 평가해 method 간 차이가 희미해짐. Old는 artistic/sketch nudity까지 엄격하게 unsafe로 잡아 SAFREE vs Ours 격차가 훨씬 명확함.

Dataset Update: I2P → MJA

I2P는 SD v1.4 baseline이 이미 너무 안전 → headroom이 없음

I2P의 한계
• 2022년 초창기 SDv1.4 공개 직후의 unsafe prompt 모음
• 현재 기준으로는 대부분 이미 안전하게 생성됨
• Baseline 자체가 거의 완벽 → method 간 구분 불가
• "Ceiling effect" — 실제 방어력 측정에 부적합
MJA = Metaphor-based Jailbreak Attack
• 은유/우회 표현으로 unsafe 이미지 유도
• 직접적인 unsafe keyword 없음 → text-only defense 우회
• 4 concepts (sexual / violent / disturbing / illegal)
• Baseline이 실제로 깨지는 adversarial prompt
Takeaway: 이미 잘 방어되는 prompt가 아닌, 실제로 깨지는 adversarial prompt에 대한 방어력 비교가 훨씬 의미있음.

Backbone Extension

SD3-medium (MMDiT)

Backbone: SD3-medium

Key differences vs SD v1.4

  • MMDiT joint attention — image + text tokens이 concat되어 self-attention. Cross-attention이 아님.
  • Velocity prediction (vs SD1.4 noise/ε). Flow Match Euler scheduler.
  • Triple text encoder: CLIP-L + CLIP-G + T5-XXL.
  • 16-ch VAE, 1024px (vs 4-ch, 512px).
Adaptation: CAS formula는 그대로 (방향 기반 cos-sim은 prediction type 무관). Probe는 joint attention의 image → text sub-matrix를 hook해서 추출. Anchor inpaint 공식은 velocity space에서 동일 선형 결합.

Backbone: FLUX.1-dev & FLUX.2-klein

FLUX.1-dev (12B DiT)
Embedded guidance — `guidance_scale`이 tensor로 transformer에 직접 주입됨. 전통적 CFG (neg/pos 2-pass) 없음.
→ CAS 계산을 위해 별도 null pass를 수동으로 돌려야 함. SafeGen의 anchor 공식도 embedded 버전으로 수정.
FLUX.2-klein (4B DiT)
Mistral encoder, 전통적 CFG 복귀 → SD v1.4와 거의 동일한 SafeGen 코드로 이식.

Status

  • 1024px re-generation 진행 중 (siml-04).
  • Preliminary — FLUX.2-klein baseline RAB 74.7% (SD v1.4 baseline 22.8% 대비 훨씬 안전).
  • SafeGen 적용 결과는 다음 미팅에 공개.
  • FLUX.1-dev는 embedded guidance 때문에 call 수가 baseline 1 → SafeGen 3–4로 증가.
Limitation — FLUX 내장 Safety Tuning:
FLUX.2-klein 모델 자체에 이미 safety tuning이 내재됨 (Flux2KleinPipeline에 safety_checker 파라미터 자체가 없음). 외부에서 끌 수 없고 weights 내부에 포함.
→ Baseline SR이 원본부터 매우 높음 (RAB 74.7%, MMA 98.1%) → unlearning method의 marginal gain이 제한적.
→ 우리 방법이 더 빛나는 건 safety-tuning 없는 SD v1.4 계열. FLUX 결과는 cross-backbone generality 입증용.
Message: SD1.4 (UNet + ε) → SD3 (MMDiT + velocity) → FLUX (DiT + flow/embedded) 까지 동일 When-Where-How 프레임워크가 작동 — method-agnostic design이 입증됨.
