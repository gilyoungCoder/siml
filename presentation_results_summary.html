WHEN — CAS (Concept Alignment Score)
At each denoising step, measure how close the current generation is to the unsafe concept. Guidance is only applied when CAS exceeds a threshold.
dprompt = εprompt − ε∅   ,   dtarget = εtarget − ε∅
CAS = cos(dprompt, dtarget)  >  τ
WHERE — Spatial Masking
Determine which pixels in the latent are generating unsafe content. Different versions use different signals: cross-attention maps, noise direction similarity, or exemplar directions.
HOW — Guidance Strategy
Modify the UNet's noise prediction only in masked regions. Core idea: subtract the "nudity direction" and/or steer toward a "safe anchor direction" in the noise prediction space.
εfinal = εcfg − st · M · dtarget + sa · M · danchor
Evaluation
NudeNet Unsafe% ↓  |  Qwen3-VL SR% ↑
SR = (Safe+Partial) / Total × 100

Background: Standard CFG in Stable Diffusion

UNet Denoising Loop (50 DDIM steps)
At each timestep t, the UNet predicts noise ε from the current noisy latent zt. The predicted noise is used to compute zt-1.
Standard CFG (2 UNet passes per step):

ε∅ = UNet(zt, t, ∅)   ← unconditional pass
εprompt = UNet(zt, t, cprompt)   ← text-conditioned pass

εcfg = ε∅ + scfg · (εprompt − ε∅)     (scfg = 7.5)
Our method adds extra UNet passes to compute concept directions:
εtarget = UNet(zt, t, c"nude person") → nudity direction dtarget = εtarget − ε∅
εanchor = UNet(zt, t, c"clothed person") → safe direction danchor = εanchor − ε∅
These directions tell us "which way is nudity" and "which way is clothed" in the noise prediction space at each denoising step.

Method Evolution

v3 → v7: How each version changed WHEN / WHERE / HOW

v3 Cross-Attention Map + SLD/DAG Guidance

WHEN: Global CAS (τ = 0.3)

CASt = cos(dprompt, dtarget)
trigger if CASt > 0.3 (sticky: once on, stays on)

WHERE: Cross-Attention Maps

Hook into UNet's cross-attention layers (attn2) at 16×16 and 32×32 resolutions.

During the εprompt forward pass, extract attention weights:
Attn = softmax(Q · KT / √d)
where Q = spatial features, K = text token embeddings.

Average attention for target tokens ("nude", "person") across heads and layers → upsample to 64×64 → normalize to [0,1].

Binary mask: M[h,w] = 1 if CAMtarget[h,w] − CAManchor[h,w] > 0.3

HOW: SLD / DAG Guidance

SLD mode:
εsafe = εcfg − s · W · dtarget

DAG adaptive mode:
area = ΣM / |M|
T(A) = 1 + 4 · W  (magnitude scaling)
εsafe = εcfg − s · (5·area) · T(A) · M · dtarget

Dual mode:
εsafe = εcfg − s · W · (dtarget − danchor)
UNet calls: ε∅ + εprompt + εtarget + εanchor  = 4 passes/step

v3 Results (Ring-A-Bell)

Key Finding
DAG (Direction-Aware Guidance) vastly outperforms SLD. Area-adaptive scaling prevents over-suppression on small nudity regions while being aggressive on large ones.
What it modifies in UNet output
The noise prediction εcfg is the UNet's final output that determines the next latent zt-1. v3 subtracts the target concept's direction vector from this output, weighted by the cross-attention spatial mask. This steers the denoising away from nudity only in pixels where the UNet is attending to nudity-related tokens.
Limitation
Cross-attention maps require a dedicated UNet forward pass for target and anchor concepts. 4 UNet calls per step. Binary thresholding of attention maps can be noisy.

v4 Spatial CAS + Anchor Inpainting

WHERE changed: Noise-Based Spatial CAS

Instead of hooking into cross-attention layers, v4 computes per-pixel cosine similarity directly in the noise prediction space.

Problem: raw 4-channel latent vectors are too noisy for pixel-level cosine similarity.
Solution: 3×3 neighborhood pooling via unfold → 36-dim vectors.
dprompt = Unfold3×3(εprompt − ε∅) ∈ ℝ36 × H·W
dtarget = Unfold3×3(εtarget − ε∅) ∈ ℝ36 × H·W

SpatialCAS[h,w] = cos(dprompt[:,hw], dtarget[:,hw])

Soft mask: M = GaussBlur(σ(10 · (SpatialCAS − 0.3)))

Sigmoid + Gaussian blur produces smooth [0,1] mask instead of binary threshold.

HOW changed: Anchor Inpainting

Anchor Inpaint (new default):
εanchor_cfg = ε∅ + scfg · danchor
εfinal = εcfg · (1 − s·M) + εanchor_cfg · (s·M)
Blends toward "clothed" generation in masked pixels

Hybrid (subtract + add):
εfinal = εcfg − st·M·dtarget + sa·M·danchor
Simultaneously push away from nudity AND toward clothed

Projection (new):
proj = (dprompt·dtarget / ||dtarget||²) · dtarget
dsafe = dprompt − s · proj
Surgically remove nudity component from prompt direction
UNet calls: ε∅ + εprompt + εtarget + εanchor  = 4 passes/step

v4 Results (Ring-A-Bell)

What changed vs v3
WHERE: Cross-attention maps → Noise-based spatial CAS
Mask type: Binary threshold → Soft sigmoid + Gaussian blur
HOW: Added anchor inpainting (blend toward safe) and projection (remove nudity component)
CAS τ: 0.3 (same as v3)
Issue: Spatial CAS is noisy
Per-pixel cosine similarity in 64×64 latent space, even with neighborhood pooling, produces less semantically meaningful masks than cross-attention. The baseline unsafe% (23.4% with anchor inpainting) is lower than v3 baseline (75.6%) because the spatial CAS itself partially works, but fine-grained masking accuracy is poor.
Projection mode introduced
First version to implement "nudity direction projection" — removing just the nudity-aligned component from dprompt while preserving the rest. This idea carries through to v6, v7, and beyond.

v6 Cross-Attention Probing

WHERE changed: Attention Probing (Zero Overhead)

v3's cross-attention required a dedicated UNet forward pass for the target concept. v6 solves this with probing:

Step 1 (offline): Pre-compute target concept's K (key) projections
   Ktarget = WK · TextEnc("nude person")

Step 2 (runtime): During the normal εprompt forward pass, hook into cross-attention layers and extract Q (query) features from the hidden states.

Step 3: Compute attention without extra UNet call:
   Attn = softmax(Qspatial · KtargetT / √d)
   → spatial mask of nudity-related pixels

This piggybacks on the existing CFG pass — zero extra UNet calls for WHERE detection.

Probing vs v3 Cross-Attention

v3: Run UNet(zt, ctarget) → extract full attention
  Cost: 1 extra UNet pass

v6: During UNet(zt, cprompt), hook Q from attn2 layers
  Compute: softmax(Q · KtargetT / √d)
  Cost: matrix multiply only (negligible vs UNet pass)

HOW: Same as v4

All guidance modes from v4: hybrid (recommended), SLD, anchor inpainting, projection. CAS threshold raised to τ = 0.6 for fewer false positives.
UNet calls: ε∅ + εprompt + probe + εtarget + εanchor  = 4 passes (WHERE is free)

v6 Results (Ring-A-Bell)

What changed vs v4
WHERE: Noise-based spatial CAS → Cross-attention probing
Mask type: Semantically meaningful token-to-pixel alignment
CAS τ: 0.3 → 0.6 (reduces false positives on benign prompts)
HOW: Same as v4 (hybrid mode recommended)
Highest SR achieved
Cross-attention probing provides semantically precise masks. The mask highlights exactly the pixels where the UNet is generating nudity-related content (via token-to-pixel alignment), rather than relying on noisy cosine similarity in latent space. This leads to 93.4% SR — the best semantic preservation of any version.
Limitation
Nudity removal weaker than v3 DAG (16% vs 0.95% unsafe). Semantic precision means smaller mask area → less aggressive removal. Still needs online target UNet pass for CAS.

v7 GLASS Exemplar-Based Guidance

Core Innovation: Pre-Computed Anchor

v3–v6 compute danchor online via a UNet pass with c"clothed person" at every step.

v7 replaces this with offline pre-computed directions from 16 diverse exemplar image pairs (nude ↔ clothed):

Offline (once):
For each exemplar pair (nude prompt, clothed prompt):
   Run full 50-step denoising with same seed
   At each timestep t, store:
     dtarget[t] = εnude − ε∅   [1,4,64,64]
     danchor[t] = εclothed − ε∅   [1,4,64,64]
   Average across 16 exemplar pairs

Runtime: Hybrid Exemplar Blend

Exemplar mode (3 UNet passes):
danchor = precomputed[t]  (no UNet call!)
εanchor = ε∅ + danchor

Hybrid blend mode:
dtarget = w · dtarget_exemplar[t] + (1−w) · dtarget_online
Blend pre-computed generic direction with image-specific online direction

Guidance (same hybrid formula):
εfinal = εcfg − st·M·dtarget + sa·M·danchor
UNet calls: ε∅ + εprompt + εtarget + εanchor  = 3 passes (~25% faster)

v7 Spatial Mask & Exemplar Details

WHERE: Spatial CAS with Exemplar Directions

Exemplar mode:
Uses online dtarget for spatial CAS (image-specific)
SpatialCAS[h,w] = cos(Unfold(dprompt), Unfold(dtarget_online))

Hybrid blend mode:
dtarget_blended = w · dtarget_exemplar[t] + (1−w) · dtarget_online
SpatialCAS[h,w] = cos(Unfold(dprompt), Unfold(dtarget_blended))

M = GaussBlur(σ(10 · (SpatialCAS − 0.3)))

Why Exemplar Works

The anchor direction from real exemplar pairs captures a data-driven "clothed" concept, more robust than a single text prompt "clothed person". Averaging over 16 diverse pairs smooths out image-specific noise.

Key Takeaways

Summary & Tradeoffs

Best Nudity Removal

v7 Exemplar Hybrid — 5.06% unsafe
Pre-computed anchor from 16 exemplar pairs + hybrid guidance.
CAS threshold tuning → 0.95% achievable.

Best Fidelity (SR + Unsafe)

v3 DAG s=5 — 0.95% unsafe, 90.5% SR
Best balance of removal and preservation.
But: requires 4 UNet passes with online target.

Best Semantic Preservation

v6 CrossAttn ts20_as20 — 93.4% SR
Precise token-to-pixel masking preserves non-nudity content.
But: 16.1% unsafe (weaker removal).

Core Tradeoff

Unsafe% ↓ ↔ SR% ↓
Aggressive guidance removes nudity but also degrades prompt fidelity.
v7 ts15: 5% unsafe / 54% SR
v7 ts25: 4.4% unsafe / 29% SR

Failed Approaches

v8 (projection) & v9 (substitution)
Projecting onto or substituting with exemplar subspace
→ loses image-specific information → baseline-level performance

Remaining Challenge

MMA adversarial prompts
57.5% unsafe even with v7. CAS-based detection fails on obfuscated prompts that bypass cosine similarity.

Next Steps

1. Improve SR-Unsafe Tradeoff
Combine v7's strong removal with v6's high SR — e.g., cross-attention WHERE + exemplar HOW
2. Adversarial Robustness
MMA dataset — need CAS detection robust to obfuscated/adversarial prompts
3. Evaluation Expansion
VQAScore alignment / FID / other unsafe categories
